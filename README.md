# 🛡️ Insurance-RAG｜保险业务员 AI 辅助系统

> **一句话定位**：
> 👉 一个 **面向保险业务员的 AI 效率工具**，通过 RAG 技术，把复杂冗长的保险条款，转化为 **可直接使用的结构化信息 + 销售话术**。

---

## 🎯 一、项目背景（Why）

### 👤 目标用户

* 保险业务员 / 销售人员
* 需要频繁向客户解释保险条款、保障范围和免责内容

### 😖 核心痛点

* 📄 条款内容冗长、专业术语多，人工查阅效率低
* 🧠 新业务员难以快速理解产品、形成稳定话术
* 🔁 不同业务员讲解不一致，影响客户信任与转化

### 🎯 产品目标

> **让业务员“不用完整读条款，也能把保险讲清楚”**

---

## 🧩 二、核心功能（What）

| 功能模块         | 说明                                            |
| ------------ | --------------------------------------------- |
| 🔍 语义检索      | 基于 Supabase 向量数据库 + OpenAI Embedding，精准定位相关条款 |
| 🧱 结构化信息抽取   | 自动提取：产品概览、核心保障、免责条款、适合人群                      |
| 🗣️ 销售话术生成   | 自动生成 2–5 条可直接使用的解释 / 劝服话术                     |
| 📌 信息来源标注    | 每条结论标注对应条款来源，增强业务员信任感                         |
| 📤 一键复制 / 导出 | 支持复制话术、导出 PDF，用于真实销售场景                        |

---

## 🔄 三、典型使用流程（How）

1️⃣ 业务员输入关于某一保险产品的问题（如保障内容、免责说明）
2️⃣ 系统通过向量检索，定位最相关的条款内容
3️⃣ AI 对条款进行结构化整理
4️⃣ 自动生成可直接对客户使用的销售话术
5️⃣ 同时标注信息来源（条款位置），便于核验

---

## 🚧 四、能力边界（非常重要）

### ✅ 当前可以做到

* 将保险条款转化为 **结构化知识 + 可用话术**
* 明显减少业务员翻阅文档的时间成本
* 作为 **业务辅助工具** 提升销售效率

### ❌ 当前刻意不做

* 不支持普通用户直接提问（如“我适合买什么保险”）
* 不做多保险产品自动对比与推荐
* 不做投保决策替代（避免 AI 误导）

> 🧠 **产品取舍原则**：
> 本项目定位为 **B 端效率工具**，而非决策系统，避免 AI 替代人类判断。

---

## 🛠️ 五、技术方案（简要）

* 🧠 LLM：OpenAI GPT 系列
* 🧬 Embedding：OpenAI Embedding API
* 🗄️ 向量数据库：Supabase
* 🚀 后端服务：FastAPI

### 🤔 为什么选择 RAG 而不是微调？

* 🔄 保险条款更新频繁，RAG 只需替换文档
* 💰 成本更低、上线更快
* 🧪 更适合产品验证与早期落地阶段

---

## 🌟 六、产品价值（So What）

* ⏱️ 帮助业务员 **快速理解** 复杂保险产品
* 🧩 输出 **标准化、可复用** 的销售话术
* ⚙️ 将 AI 能力真正嵌入业务工作流，而不是停留在 Demo
