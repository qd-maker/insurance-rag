# Insurance-RAG｜保险条款智能问答产品（AI 产品向）

## 一、项目背景（Why）

**用户痛点**：

* 保险条款内容冗长、专业术语多，普通用户难以快速找到关键信息
* 客服或业务人员需要频繁翻阅 PDF / 文档，效率低、容易出错

**产品目标**：

> 打造一个「可以直接提问的保险助手」，让用户用自然语言快速获取保险条款中的准确信息。

---

## 二、产品方案（What）

### 核心功能

* 📄 支持导入保险条款文档（PDF / 文本）
* 🔍 基于语义检索，从条款中定位最相关内容
* 🤖 结合大语言模型，生成自然语言回答
* 📌 回答基于原文内容，降低“胡编”的风险

### 典型使用场景

* 用户提问：**“理赔需要准备哪些材料？”**
* 系统行为：

  1. 从保险条款中检索相关段落
  2. 将检索结果作为上下文
  3. 由大模型生成结构化回答

---

## 三、技术方案（How）

### 整体流程

1. 文档解析：将保险条款拆分为可检索文本块
2. 向量化：使用 Embedding 模型生成向量表示
3. 向量检索：根据用户问题匹配最相关条款
4. RAG 生成：将检索结果输入大模型生成回答
5. API 服务：通过 FastAPI 对外提供问答接口

> 选择 RAG 而非微调的原因：
>
> * 保险条款经常更新，RAG 可直接替换文档
> * 成本更低、上线更快、更适合产品原型阶段

---

## 四、产品视角的关键取舍（面试重点）

* ❓ **为什么不用模型微调？**

  * 当前阶段更关注“可用性”而非极致效果
  * RAG 更灵活，适合业务频繁变更的场景

* ⚠️ **如何应对大模型不确定性？**

  * 限定回答必须基于检索内容
  * 不命中时返回“未找到相关条款”提示

* 📈 **后续可优化方向**：

  * 多文档支持（多保险产品）
  * 检索召回率优化
  * 引入人工校验或高频问答缓存
